III. Lộ Trình Học DevOps (6 Tháng - 24 Tuần, 600 Giờ)

Giai Đoạn 1: Nền Tảng Cơ Bản (4 Tuần, 100 Giờ)

  • Khóa học: Introduction to DevOps (IBM) + IBM DevOps and Software Engineering (Khóa 1-3)
  • Tuần 1 (25 giờ): Giới thiệu DevOps + Git
    • Lý thuyết: Văn hóa DevOps, CI/CD (9 giờ).
    • Thực hành: Cài Git, tạo repository GitHub (16 giờ).
  • Tuần 2 (25 giờ): Linux cơ bản
    • Lệnh cơ bản (ls, cd, chmod), quản lý file (10 giờ).
    • Shell scripting đơn giản (15 giờ).
  • Tuần 3-4 (50 giờ): Python cho DevOps
    • Cú pháp Python, xử lý file (20 giờ).
    • Script tự động hóa (backup file) (30 giờ).

Giai Đoạn 2: Công Cụ DevOps Cốt Lõi (8 Tuần, 200 Giờ)

  • Khóa học: DevOps on AWS Specialization (Khóa 1-2) + IBM DevOps and Software Engineering (Khóa 4-6)
  • Tuần 5-6 (50 giờ): CI/CD với Jenkins/AWS
    • Cài Jenkins, tạo pipeline cơ bản (20 giờ).
    • AWS CodePipeline, CodeDeploy (30 giờ).
  • Tuần 7-10 (100 giờ): Docker
    • Docker cơ bản: Container là gì, cài Docker (30 giờ).
    • Build image, chạy container (40 giờ).
    • Dự án: Container hóa ứng dụng web đơn giản (30 giờ).
  • Tuần 11-12 (50 giờ): Kubernetes cơ bản
    • Cụm Kubernetes, pod, deployment (25 giờ).
    • Triển khai ứng dụng trên Minikube (25 giờ).

Giai Đoạn 3: Thực Hành Nâng Cao (8 Tuần, 200 Giờ)

  • Khóa học: IBM DevOps and Software Engineering (Khóa 7-9) + Continuous Delivery & DevOps
  • Tuần 13-16 (100 giờ): Ansible + Terraform
    • Ansible: Cấu hình server tự động (50 giờ).
    • Terraform: Infrastructure as Code trên AWS (50 giờ).
  • Tuần 17-20 (100 giờ): Monitoring + Security
    • Cài Prometheus, Grafana để giám sát (40 giờ).
    • Bảo mật pipeline (IAM, secret management) (30 giờ).
    • Dự án: Pipeline hoàn chỉnh với monitoring (30 giờ).

Giai Đoạn 4: Chuẩn Bị Xin Việc (4 Tuần, 100 Giờ)

  • Tuần 21-24 (100 giờ):
    • Ôn lý thuyết: CI/CD, IaC, container (25 giờ).
    • Dự án: Xây pipeline hoàn chỉnh (50 giờ):
      • GitHub → Jenkins → Docker → AWS + giám sát Grafana.
    • Chuẩn bị phỏng vấn: Ôn câu hỏi DevOps (25 giờ).
    • Đẩy code lên GitHub, viết README chi tiết.
